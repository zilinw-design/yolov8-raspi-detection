## 未解决的核心问题

### 问题1：复杂环境下轮廓检测不可靠

**现象**：摄像头对着复杂桌面/房间时，`findContours` 找到的最大轮廓是房间暗背景（面积 100 万+ 像素），而不是目标图形。

**已做的**：贴边过滤 —— 排除紧贴画面边缘的大面积轮廓。

**还差的**：如果暗背景不贴边（例如画面中有大面积暗色家具），贴边过滤无效。

### 问题2：屏幕显示测试图形时检测不到

**现象**：屏幕全屏显示黑色正方形（test_shapes.html），摄像头对着屏幕，无检测结果。

**可能原因**（待确认）：
- 屏幕刷新率与摄像头快门不同步产生摩尔纹
- 屏幕的"黑色"在相机里不是纯黑（背光泄漏）
- OTSU 阈值在屏幕画面和房间背景混杂时不稳定
- 屏幕外框/边框被检测为轮廓

### 问题3：相机参数对环境敏感

**已认知**：不可能通过静态参数适配所有光照条件。这是物理限制。

### 问题4：屏幕画面和真实 A4 纸是两种不同场景

| | 屏幕测试 | A4 纸测量 |
|---|---|---|
| 光源 | 屏幕自发光 | 环境光反射 |
| 黑色 | RGB(0,0,0) ≈ 完美黑 | 油墨/打印黑 |
| 动态范围 | 极高（背光） | 取决于环境光 |

为屏幕测试调的参数，A4 纸场景不一定适用。

## 已解决（2026-07-20）

### 检测算法：文档扫描管线

**方案**：Canny → findContours → approxPolyDP 找4边形 → 透视矫正 → OTSU 检测图形

| 组件 | 作用 | 依据 |
|---|---|---|
| `find_document_region()` | 找A4纸/屏幕的4边矩形 | OpenCV Document Scanner 标准管线 |
| 宽高比验证 (0.9~2.5) | 过滤非矩形杂物 | A4=1.414, 屏幕=1.78 |
| 角点内缩黑边框验证 | 4角点向重心缩5%→采样→必须全黑 | 参考赛题方案：防误识别窗户/显示器 |
| `four_point_transform()` | 透视矫正→正视图 | 背景自动裁掉 |
| OTSU + findContours (正视图内) | 检测黑色图形 | 白底+黑图形=双峰直方图=最优 |

### 对焦：自动锁定

开自动对焦→1秒→读当前值→锁定。v4l2-ctl 验证通过。

### 相机参数：基准确定

brightness=10, contrast=35, saturation=80, sharpness=2, backlight_compensation=36, white_balance_automatic=1
